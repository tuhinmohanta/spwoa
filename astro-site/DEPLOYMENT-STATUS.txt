===============================================================================

   git commit -m "Deploy optimized Astro site to GitHub Pages"

3. Push to trigger deployment:

- Build your Astro site
- Deploy to GitHub Pages

TROUBLESHOOTING
===============================================================================

If deployment fails:

1. CHECK WORKFLOW LOGS
   - Go to repository → Actions tab
   - Click on the failed run
   - Expand failed steps to see error messages

2. COMMON ISSUES:
   
   a) Build fails with "npm ci" error
      Solution: Ensure package-lock.json is committed
      
   b) Node modules not found
      Solution: Workflow already handles this with npm ci
      
   c) Permission denied
      Solution: Workflow already has correct permissions
      
   d) Wrong files deployed
      Solution: Workflow deploys from astro-site/dist/

3. VERIFY GITHUB PAGES SETTINGS
   Repository → Settings → Pages:
   - Source: Deploy from a branch
   - Branch: gh-pages (created automatically)
   - Folder: / (root)
